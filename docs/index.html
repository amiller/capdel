dynamic capability delegation for agents

A capability minted the moment you dispatch a subagent — not picked from a menu of scopes.

Secret managers and OAuth scopes already keep static, pre-defined slices of authority in the right hands. Agent work isn't static — the slice a subagent needs is known only at dispatch: this directory, these two commands, the next 30 minutes. capdel lets a trusted agent attenuate its own authority to exactly that, hand over only a token, and enforce it from a local broker. Escalate mid-task, watch the whole tree, revoke a branch and everything under it.

$ python3 capdel.py serve read the source →

The whole idea in one screen. You give a helper a read-only key. It hits a wall, asks, you approve, it finishes — and it never once held more than you granted.

helper — holds one token, nothing else
# you mint a read-only capability — it prints an id + a token (the "ticket")$ capdel mint fs --root ~/work --ops list,read --ttl 30m --name helper id=cap-e0ce… token=ct-3471… # hand the helper only these # the helper reads — works→ invoke {"op":"list","path":"~/work"} ALLOW {"entries":[{"name":"report.txt","type":"file","size":32}]} # it tries to WRITE — refused, with the exact reason→ invoke {"op":"write","path":"~/work/summary.txt"} DENY {"violated":"op 'write' not in granted ops ['list','read']"} # so it ASKS — sending only the delta, not the whole scope→ escalate {"want":{"ops":["list","read","write"]},"reason":"write summary"} {"request_id":"req-e76f…","status":"pending"} # you rule on it — a fresh, tighter grant$ capdel approve req-e76f… --ttl 20m # → new token + cap→ invoke {"op":"write","path":"~/work/summary.txt"} ALLOW {"written":40,"created":true} # the original token still can't write. take it all back, anytime:$ capdel revoke cap-e0ce… # kills it and its whole subtree now
where it fits

Scoped access is solved — for scopes you can name in advance.

Secret managers, OAuth scopes, workload identity, offline capability tokens like UCAN and Biscuit — the ecosystem is full of ways to keep a pre-defined slice of authority in the right hands. capdel doesn't replace any of them. It covers the part they don't: the slice that only exists at dispatch, when a trusted agent works out what a riskier one needs and mints exactly that.

OAuth scopes / IAM roles — a fixed menu decided up front Attenuate any capability on the spot: a subset of ops, a path prefix, a shorter clock. No standing switchboard of thousands of scopes.
UCAN / Biscuit — attenuable tokens, verified offline at each resource Same attenuation-first idea, kept online in one local broker — a smaller trusted base, and it brokers files, commands and sockets, not only API calls.
A secret manager hands out the secret for the holder to use The broker keeps the authority; the holder gets only a token — and you revoke a branch and everything under it at once, or tie a grant to its reason so it dies when the reason does.
the moves

Six verbs. One reference monitor.

The broker is the only thing that enforces anything. Everything a holder can do is one of these — checked structurally, logged, revocable.

mint
operator · CLI

Create root authority from nothing. The token is shown once; the broker keeps only its hash.

attenuate
any holder

Mint a strictly narrower child. The subset is checked structurally — ask for more and it's a 403.

invoke
any holder

Exercise the capability — read a file, run an allowlisted command, open one brokered connection, call a wrapped model.

escalate
any holder

Ask for more, with a reason. You rule via approve / deny; the requester polls and gets a fresh grant.

revoke
operator · CLI

Kill a capability and its whole subtree — immediately, checked at every invoke.

event
operator · CLI

Fire a trusted event; every cap that declared closes-on it auto-revokes. Authority dies with its reason.

what you can grant

Four kinds of authority. Each one narrow.

fslist · read · write · stat

Read or write inside one directory. Nothing above the root, only the ops you named.

execargv prefixes

Run allowlisted commands — argv prefixes only. No shell, no substitution, no surprises.

nethost:port

One brokered TCP connect to an allowlisted host and port. No open socket is ever handed out.

llmmodels · max-tokens

Call a shared model key the broker injects. The holder sends messages and never sees the key.

why it holds

The details that make it safe to hand out.

Self-describing tokens

A holder bootstraps from GET /caps/<id>: its exact constraints, literal usage examples, and how to ask for more. Nothing to document or pass along.

Denials name the rule

A refusal comes back with the constraint it violated — so an agent can decide to escalate instead of failing blind.

Proof-of-possession

Opt into a token that's an HMAC key, never sent on the wire. A captured request is useless — the signature binds method, path, body, a one-time nonce, and time.

Closure

Tie a grant to its reason, not just a clock. --closes-on build-passed → the moment you file the event, the cap and its children are gone.

Dial-out for remote

Worker on another machine? The broker dials out to a rendezvous — no inbound port, and your authority never leaves your laptop.

The grant tree

capdel tree is your live exposure map; capdel audit logs every allow and deny. You can always see exactly what's out there.

get it

One file. Stdlib-only Python 3.9+.

No dependencies, no build. The broker holds authority on 127.0.0.1; state lives in ~/.capdel.

# clone and run the doorman git clone https://github.com/amiller/capdel && cd capdel python3 capdel.py serve & # broker on 127.0.0.1:4571 # mint a root capability — hand a subagent only the token it prints python3 capdel.py mint fs --root ~/work --ops list,read --ttl 30m --name helper python3 capdel.py tree # see everything you've granted
